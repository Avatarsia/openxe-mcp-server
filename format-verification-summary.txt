1. TABLE FORMAT
   - Status: WORKING
   - Structure: Markdown table with pipe delimiters

   - Default columns: All fields from the endpoint
   - Fields parameter: Parsed but currently returns all columns in table
   - Example endpoint: openxe-list-addresses
   - Sample size: 4.973 bytes (50 records)
   - Characteristics:
     * Header row with column names
     * Separator row with dashes
     * Data rows with pipe delimiters

     * Preserves German special characters (ä, ö, ü, etc.)

2. CSV FORMAT
   - Status: WORKING
   - Structure: Semicolon-delimited format
   - First row: Headers (column names)
   - Data rows: Field values separated by semicolons
   - Default columns: All fields from the endpoint
   - Example endpoint: openxe-list-orders
   - Sample size: 3.418 bytes (50 records)
   - Characteristics:
     * Uses semicolon as delimiter (not comma)
     * Header row included
     * No quote wrapping for simple values
     * Handles German text correctly
     * Clean, parseable format

3. IDS FORMAT
   - Status: WORKING
   - Structure: Comma-delimited list of numeric IDs
   - No header row
   - Only contains numeric IDs
   - Example endpoint: openxe-list-addresses
   - Sample size: 181 bytes (50 IDs)
   - Characteristics:
     * Pure numeric output (e.g., "29,33,1,239,120,...")
     * Comma-delimited with no spaces
     * No header or metadata
     * Very compact format
     * Useful for ID-only queries

4. JSON FORMAT (Default)
   - Status: WORKING
   - Structure: JSON with metadata wrapper
   - Top-level keys: _info, _hint, data
   - Data array: Contains full record objects
   - Sample size: 14.441 bytes (50 records)
   - Characteristics:
     * Metadata fields: _info (result info), _hint (usage hint)
     * Full data structures preserved
     * Truncation warning if list exceeds limit
     * Filter information in _info
     * Standard 2-space JSON indentation
